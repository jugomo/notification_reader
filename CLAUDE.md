## Firebase Setup (required before first run)

This project requires Firebase. After cloning or adding new Firebase features, run:

```bash
# Install CLIs (once)
npm install -g firebase-tools
dart pub global activate flutterfire_cli

firebase login
flutterfire configure   # generates lib/firebase_options.dart and android/app/google-services.json
flutter pub get
```

In Firebase Console, enable:
- **Authentication** → Email/Password provider
- **Realtime Database** → create database (start in test mode for development)

## Architecture

This is a Flutter + Android hybrid app. Flutter handles the UI and Firebase; native Kotlin handles OS-level notification access (which Flutter cannot do directly).

### Flutter → Android bridge

`MainActivity.kt` exposes a `MethodChannel` on `com.example.notification_reader/service` with four methods:

| Method | Action |
|---|---|
| `hasPermission` | Checks `NotificationListenerService` grant via `Settings.Secure` |
| `openSettings` | Opens `ACTION_NOTIFICATION_LISTENER_SETTINGS` |
| `startService` | Starts `LockScreenService` as a foreground service |
| `stopService` | Stops `LockScreenService` |

`lib/main.dart` calls these via `static const _channel = MethodChannel(...)`.

### Native Android services

- **`NotificationReaderService`** — `NotificationListenerService` subclass. Holds a singleton `instance` so other components can call `getCurrentNotifications()`.
- **`LockScreenService`** — Foreground service that posts a persistent, lock-screen-visible notification with a "🔊 Leer notificaciones" action button. Requires `FOREGROUND_SERVICE_SPECIAL_USE`.
- **`NotificationActionReceiver`** — `BroadcastReceiver` triggered by the lock-screen button. Reads active (non-ongoing) notifications through `NotificationReaderService.instance` and speaks them aloud via Android `TextToSpeech` in `es_ES`.

### Firebase integrations

- **Auth** (`firebase_auth`): Email/password. Auth state drives routing via `StreamBuilder<User?>` on `FirebaseAuth.instance.authStateChanges()` in `NotificationReaderApp`. Unauthenticated → `AuthScreen`; authenticated → `HomePage`.
- **Realtime Database** (`firebase_database`): All notifications are written to `users/{uid}/notifications/{auto-id}` with fields `source` ('system' | 'fcm'), `appName`, `packageName`, `title`, `body`, `receivedAt` (ISO string). User profile (email, uid) is written to `users/{uid}/profile` on login.
- **Cloud Messaging** (`firebase_messaging`): Three delivery paths all funnel through `_saveNotification()`:
  - Foreground: `FirebaseMessaging.onMessage`
  - Background: `_firebaseMessagingBackgroundHandler` (top-level, `@pragma('vm:entry-point')`)
  - Terminated: `getInitialMessage()`

### Key files

- [lib/main.dart](lib/main.dart) — App entry point, Firebase init, FCM handlers, all UI widgets
- [lib/auth/auth_screen.dart](lib/auth/auth_screen.dart) — Login/signup screen (toggles between modes)
- [lib/firebase_options.dart](lib/firebase_options.dart) — Generated by `flutterfire configure`, excluded from git (run `flutterfire configure` locally to regenerate)
- [android/app/src/main/kotlin/.../MainActivity.kt](android/app/src/main/kotlin/com/example/notification_reader/MainActivity.kt) — MethodChannel bridge
- [android/app/src/main/AndroidManifest.xml](android/app/src/main/AndroidManifest.xml) — Declares all services, receivers, and permissions
